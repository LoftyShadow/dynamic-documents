# 🚀 快速参考 (v2.0)

## 配置文件

所有配置都在 `docs-config.json` 中:

```json
{
  "projects": [
    {
      "name": "项目名称",
      "sourceDir": "C:\\文档路径",
      "enabled": true,
      "collapsed": false
    }
  ]
}
```

## 一键启动

```bash
pnpm run dev
```

启动:
- 📁 多项目文件监控
- 🌐 开发服务器(http://localhost:5173)

## 常用命令

| 命令 | 说明 |
|------|------|
| `pnpm run dev` | 启动监控 + 开发服务器 ⭐ |
| `pnpm run watch` | 仅启动文件监控 |
| `pnpm run generate` | 手动生成侧边栏 |
| `pnpm run check` | 检查项目状态 |
| `pnpm run docs:dev` | 仅启动 VitePress |

## 配置项目

### 1. 添加新项目

编辑 `docs-config.json`:

```json
{
  "projects": [
    {
      "name": "新项目",
      "sourceDir": "C:\\新路径",
      "enabled": true,
      "collapsed": false
    }
  ]
}
```

### 2. 禁用项目

设置 `enabled: false`:

```json
{
```

## 侧边栏结构

生成的侧边栏顺序:

```
1. 📚 本地通用文档 (localFixedDocs)
   └── 项目根目录的通用文档

2. 📁 项目文档 (projects)
   ├── 项目 1
   ├── 项目 2
   └── 项目 3

3. 🔗 固定链接 (fixedLinks)
   └── 外部链接
```

## 工作流程

1. **编辑配置文件** `docs-config.json`
2. **重启服务** `pnpm run dev`
3. **在源目录中编辑文档**
4. **自动同步和更新**
5. **浏览器自动刷新**

## 注意事项

⚠️ **标题格式**: `# 标题` (井号后有空格)

⚠️ **文件编码**: 使用 UTF-8

⚠️ **至少启用一个项目**: 必须有 `enabled: true`

## 故障排除

**配置不生效?**
```bash
# 重启服务
Ctrl+C
pnpm run dev
```

**目录不存在?**
- 检查 `sourceDir` 路径是否正确
- 确保目录存在且有读取权限

**侧边栏为空?**
- 检查项目是否启用 (`enabled: true`)
- 检查目录中是否有 `.md` 文件
- 运行 `pnpm run check` 检查状态
